# NeuG Core Source Code

This directory contains the core implementation of NeuG (Network-unified Graph), a high-performance graph database with dual-mode architecture supporting both transactional and analytical workloads.

## Directory Structure Overview

### 🏗️ [`compiler/`](./compiler/)
**Query Compilation and Planning**

The compiler module handles the complete query compilation pipeline from parsing to physical plan generation.

- **`antlr4/`** - ANTLR4-based grammar parsers for Cypher query language
- **`binder/`** - Semantic analysis and binding of query expressions to schema
- **`catalog/`** - Metadata catalog management for schemas, tables, and indexes
- **`common/`** - Common utilities and data structures for compilation
- **`extension/`** - Extension framework for custom functions and operators
- **`function/`** - Built-in function implementations (scalar, aggregate, etc.)
- **`gopt/`** - Graph optimization passes and rule engine
- **`graph/`** - Graph-specific compilation logic and optimizations
- **`main/`** - Main compiler entry points and orchestration
- **`optimizer/`** - Query optimization engine with cost-based optimization
- **`parser/`** - Query parsing and AST generation
- **`planner/`** - Logical and physical plan generation
- **`transaction/`** - Compilation-time transaction handling
- **`utils/`** - Compiler utility functions and helpers

### ⚡ [`execution/`](./execution/)
**Query Execution Engine**

The execution engine interprets and executes physical plans generated by the compiler.

- **`common/`** - Execution context, column-based data structures, type system, and base operator classes for data flow and state management
- **`execute/`** - Physical operator implementations, pipeline orchestration, and plan parsing for query execution
- **`expression/`** - Expression evaluation engine with data accessors, expression tree processing, and specialized predicate handling
- **`extension/`** - Extension framework for loading and integrating custom operators and functions
- **`utils/`** - Execution utilities including operation profiling, timing, and Protocol Buffers parsing helpers

### 🗄️ [`storages/`](./storages/)
**Storage Layer Implementation**

The storage layer manages graph data persistence, indexing, and access patterns.

- **`csr/`** - Compressed Sparse Row (CSR) format implementations
  - Immutable and mutable CSR structures
  - Adjacency list management
  - Compressed graph topology storage
- **`graph/`** - High-level graph storage abstractions
  - Property graph implementation
  - Vertex and edge table management
  - Schema definition and validation
- **`loader/`** - Data loading and ingestion modules
  - CSV and Arrow format loaders
  - Bulk loading optimizations
  - Data transformation pipelines

### 🔄 [`transaction/`](./transaction/)
**Transaction Management**

Implements both embedded (global lock) and service (MVCC) transaction models.

- **Core Transaction Types:**
  - `read_transaction.cc` - Read-only transaction implementation
  - `insert_transaction.cc` - Data insertion transaction handling
  - `update_transaction.cc` - Data modification transaction logic
  - `compact_transaction.cc` - Background compaction operations
- **`version_manager.cc`** - MVCC timestamp and version management
- **`wal/`** - Write-Ahead Logging implementation
  - WAL record management
  - Recovery and replay mechanisms
  - Durability guarantees

### 🛠️ [`utils/`](./utils/)
**Core Utilities and Infrastructure**

Common utilities and infrastructure components used throughout the system.

- **Core Utilities:**
  - `arrow_utils.cc` - Apache Arrow integration utilities
  - `file_utils.cc` - File system operations and management
  - `string_utils.cc` - String manipulation and processing
  - `yaml_utils.cc` - YAML configuration parsing
  - `pb_utils.cc` - Protocol Buffers serialization utilities
  - `service_utils.cc` - Service and networking utilities
  - `app_utils.cc` - Application lifecycle management
- **`exception/`** - Custom exception handling framework
- **`property/`** - Property type system and columnar storage
- **`proto/`** - Protocol Buffers definitions and generated code
- **`remote/`** - Remote procedure call and networking utilities

### 🚀 [`main/`](./main/)
**Main Database Engine**

Core database engine implementation and session management.

- **Core Engine:**
  - `neug_db.cc` - Main database engine implementation
  - `neug_db_session.cc` - Database session management
  - `query_processor.cc` - Query processing orchestration
  - `query_result.cc` - Query result handling and formatting
  - `connection.cc` - Client connection management
  - `file_lock.cc` - File-based locking mechanisms
- **`app/`** - Application framework and plugin system

### 🌐 [`server/`](./server/)
**Network Server Implementation**

HTTP and RPC server implementations for service mode operations.

- `brpc_http_hdl_mgr.cc` - BRPC-based HTTP service handler
- `neug_db_service.cc` - Database service implementation and API endpoints
